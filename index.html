Volt

A strictly typed, schematic-based programming language.

View on GitHub

Purpose & Philosophy

Volt replaces abstract control flow with physical hardware schematics. It forces developers to manage memory and execution paths explicitly through continuous traces. Designed for embedded systems and performance-dependent environments, Volt ensures memory safety by physically requiring all data to be routed to a terminal Ground component. If a trace is left open, the code simply will not compile.

Core Syntax

Traces ===>

The core mechanism for pushing data payloads across the board from one node to the next.

Capacitors ||[32v]

Strictly typed variables. The voltage bracket acts as a rigid byte constraint.

Resistors /\/\/\

Used to safely mutate or cast data to match the requirements of downstream components.

Ground _|_

Explicit garbage collection. Terminates traces securely to prevent compilation failure.

Sample Circuit: FizzBuzz

1v ===> ||[32v] i ===> _|_ ;

(~) [ 15v ] {
    i ===> ||[32v] tempFizz ===> Mod 3v ===> Eq 0v ===> ||[8v] isFizz ===> _|_ ;
    i ===> ||[32v] tempBuzz ===> Mod 5v ===> Eq 0v ===> ||[8v] isBuzz ===> _|_ ;

    _/_[isFizz] {
        HIGH: _/_[isBuzz] {
            HIGH: "FizzBuzz" ===> [=][PrintNode] ===> _|_ ;
            LOW:  "Fizz" ===> [=][PrintNode] ===> _|_ ;
        }
        LOW: _/_[isBuzz] {
            HIGH: "Buzz" ===> [=][PrintNode] ===> _|_ ;
            LOW:  i ===> [=][PrintNode] ===> _|_ ;
        }
    }
    
    i ===> Add 1v ===> i ===> _|_ ;
}
