Strict Traces. Explicit Grounding.

Volt

A strictly typed, schematic-inspired programming language with explicit memory lifecycle control. Reference implementation: TextX grammar and interpreter in volt-interpreter.

View interpreter on GitHub

Core Guarantees

  • Explicit allocation/deallocation: variables are created only via voltage source through typed storage into || capacitor stores and released via _|_ grounding.
  • Compile-time leak rejection: path-sensitive static analysis rejects programs where any allocated source survives termination on any control-flow path.
  • No GC dependency: the runtime does not rely on garbage collection for language memory lifecycle.
  • Physically constrained memory: integer writes respect signed/unsigned bit width, including non-byte-aligned widths such as 9v.
  • Deterministic cleanup on faults: the runtime clears remaining allocations on termination, including errors.

Core Syntax

Pipe trace

===>

Links data flow along a statement.

Typed storage

[+Nv-] / [-Nv+]

Fixed-width cell: unsigned vs signed by rail polarity.

Capacitor store

|| name

Binds or updates the named variable.

Ground

_|_

Ends the trace and deallocates what it carried.

Memory Declaration Example

"admin" ===> [+40v-] ===> || username ;

Initialize a 40-bit unsigned variable named username with the value "admin".

Sample circuit: FizzBuzz

1v ===> [-32v+] ===> || current_number ;
(~)[15v] {
    current_number ===> (%) 3v ===> (=) 0v ===> [+8v-] ===> || is_fizz ;
    current_number ===> (%) 5v ===> (=) 0v ===> [+8v-] ===> || is_buzz ;
    _/_[is_fizz] {
        [^]: _/_[is_buzz] {
            [^]: "FizzBuzz" ===> (O) ===> _|_ ;
            [v]: "Fizz" ===> (O) ===> _|_ ;
        }
        [v]: _/_[is_buzz] {
            [^]: "Buzz" ===> (O) ===> _|_ ;
            [v]: current_number ===> (O) ===> || current_number ;
        }
    }
    current_number ===> (+) 1v ===> || current_number ;
    is_fizz ===> _|_ ;
    is_buzz ===> _|_ ;
}
current_number ===> _|_ ;

Try the interpreter

Clone volt-interpreter and run programs locally.

Requirements

  • Python 3.9+
  • textX

Setup

python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install textX

Run a program

From the interpreter repo root:

cd interpreter
python3 interpreter.py ../sample_programs/hello_world.volt

Run all tests

./run_tests.sh

Repository layout

  • interpreter/volt.tx — grammar
  • interpreter/interpreter.py — execution engine + static analysis
  • sample_programs/ — example Volt programs
  • tests/test_engine.py — regression and edge cases (authoritative expected behavior)
  • run_tests.sh — one-command test runner
